# 📌 필독 (Project Notice) 및 하드웨어 최적화 안내

## 1. 프로젝트 일러두기 (Project Notice)

* **개발 목적:** 본 프로젝트는 상업적 목적이 아닌, 1인 개발자 주도의 거대 언어 모델(LLM) 기반 자율 비행 드론 제어 기술 실증(PoC, Proof of Concept) 및 독립적 연구를 목적으로 진행되었습니다.
* **검증 범위:** 상용 수준의 장기 내구성 테스트보다는, 자원이 극도로 제한된 에지 컴퓨팅 환경(Raspberry Pi 3, 1GB RAM)과 지상 관제국(GCS) 간의 3-Node 분산 제어 아키텍처를 통해, LLM 추론 결과가 지연 없이(Zero-Latency) 물리적 기동으로 변환되는 메커니즘을 구현하고 검증하는 데 초점을 맞추었습니다.
* **개발 방법론 (AI-Assisted Pair Programming):** 본 시스템의 통합 아키텍처 설계 및 소스 코드 작성의 90% 이상은 대규모 언어 모델(Google Gemini Pro)과의 페어 프로그래밍을 통해 수행되었습니다. 프로그래밍 비전공자 기준 3개월이라는 단기간 내에 이기종 통신망(ROS 2 - PX4)과 LLM을 결합한 풀스택(Full-stack) 관제 시스템을 성공적으로 구축함으로써, AI 기반 개발 프로세스의 실효성을 입증하였습니다.
* **검증 기록 및 시각 자료:** 비정형 자연어 명령의 3D 웨이포인트(Waypoint) 변환 및 MicroXRCE-DDS 기반의 실시간 통신 제어 등 시스템의 핵심 알고리즘은 가상 물리 엔진(Gazebo SITL) 및 QGroundControl 통합 환경에서 역학적 오차 없이 정상 동작함이 완벽하게 검증되었습니다.
* **라이선스 및 보존:** 본 프로젝트는 Apache 2.0 라이선스를 따르며, Zenodo(CERN)의 글로벌 오픈 액세스 저장소에 공식 등재되어 영구적인 디지털 객체 식별자(DOI)를 부여받았습니다. 해당 라이선스 규정 범위 내에서 누구나 자유로운 활용 및 변형이 가능합니다.
* **시스템 개발 환경:** 크로스 컴파일 및 시스템 호환성을 극대화하기 위해 다중 노드의 운영체제(OS) 환경을 통일하였습니다.
  * **AI 추론 및 관제 노드 (로컬 PC):** Ubuntu Linux 22.04.5 LTS (64-bit)
  * **에지 노드 (Raspberry Pi 3):** Ubuntu Linux 22.04.5 LTS Server (64-bit)
  * 기타 의존성 도구 및 패키지는 Ubuntu 표준 저장소(APT)를 통해 호환성 문제 없이 범용적으로 구축 가능합니다.

## 2. 하드웨어 시스템 최적화 필수 안내 (Raspberry Pi 3)

본 시스템의 에지 노드(Raspberry Pi 3)가 1GB RAM의 물리적 한계를 극복하고 3차원 비행 역학 제어 및 실시간 통신 스크립트를 안정적으로 연산하기 위해서는, 가상 메모리(ZRAM) 할당 및 쿨링 제어를 포함한 하드웨어 시스템 튜닝이 필수적으로 선행되어야 합니다. 

또한, 기기 발열 제어를 전제로 한 CPU 오버클럭을 강력히 권장합니다. (오버클럭 적용 시 외부 전원과 연동된 쿨링팬 장착이 필수적입니다.) 해당 하드웨어 최적화 설정에 대한 상세한 기술적 방법론은 아래의 가이드 링크를 반드시 확인하시기 바랍니다.
